SetupOracle11ginUbuntu

发表于 2016-05-24

ubuntu14.04 X64 安装oracle11g

一、安装依赖环境

1、更新apt-get

执行update更新命令：

$ sudo apt-get update
$ sudo apt-get dist-upgrade

2、安装jdk

如果已经安装过的，可以跳过这一步。
这里，我安装的是jdk7：

$ sudo apt-get install openjdk-7-jre
$ sudo apt-get install openjdk-7-jdk

还有另一种安装java的方式，就是去oracle官网下载tar.gz格式的安装包或者rpm的安装包。

3、配置java环境变量

打开/etc/profile，使用命令：sudo vi /etc/profile，在文件最后面添加如下内容：

JAVA_HOME=/usr/java/jdk1.7.0_40
CLASSPATH=.:$JAVA_HOME/lib.tools.jar
PATH=$JAVA_HOME/bin:$PATH
export JAVA_HOME CLASSPATH PATH

保存退出，使用source命令重新执行以下初始化文件

source /etc/profile

查看一下java版本，是否安装成功：

$ java -version

如果java安装成功，会显示如下内容：

java version "1.7.0_79"
Java(TM) SE Runtime Environment (build 1.7.0_79-b15)
Java HotSpot(TM) 64-Bit Server VM (build 24.79-b02, mixed mode)

4、安装依赖包（这一步很重要！）

首先修改软件源，增加源地址：

$ sudo gedit /etc/apt/sources.list

为了能顺利安装32位兼容包ia32-libs，需要在源中增加这些行(也有人说装了g++-multilib就不用考虑ia32-libs了，我没试过)： 

deb http://extras.ubuntu.com/ubuntu trusty main
deb-src http://extras.ubuntu.com/ubuntu trusty main
deb http://archive.ubuntu.com/ubuntu/ raring main restricted universe multiverse

保存源文件后退出。

下面使用apt-get进行依赖包的安装，下面的要全部装上，对后面使用图形化安装界面有影响。
这些名称都来自网上，具体为什么要装这一些依赖包，以后需要仔细研究一下！

5、配置安装前的参数

安装完依赖包之后，首先看看swap空间够不够，一般swap要设置在内存的1.5倍左右比较合适：

$free -m

然后检查系统变量：

sudo /sbin/sysctl -a | grep aio-max
sudo /sbin/sysctl -a | grep file-max
sudo /sbin/sysctl -a | grep shm
sudo /sbin/sysctl -a | grep sem 
sudo /sbin/sysctl -a | grep ip_local_port_range 
sudo /sbin/sysctl -a | grep rmem_default 
sudo /sbin/sysctl -a | grep rmem_max 
sudo /sbin/sysctl -a | grep wmem_default 
sudo /sbin/sysctl -a | grep wmem_max

编辑/etc/sysctl.conf文件

sudo gedit /etc/sysctl.conf

然后根据上面命令中得到的参数值，填写在/etc/sysctl.conf中对应的参数上

fs.aio-max-nr = 65536
fs.file-max = 400373
kernel.shmall = 18446744073692774399
kernel.shmmax = 18446744073692774399
kernel.shmmni = 4096
kernel.sem = 32000    1024000000    500    32000
net.ipv4.ip_local_port_range = 32768    61000
net.core.rmem_default = 212992
net.core.rmem_max = 212992
net.core.wmem_default = 212992
net.core.wmem_max = 212992

运行一下命令更新内核参数： 

sudo /sbin/sysctl -p

将对当前用户的内核限制，添加在/etc/security/limits.conf文件中，增加以下内容（uzipi为我的当前用户）（为什么要这样做？）：

uzipi         soft    nproc   2047 
uzipi         hard    nproc   16384 
uzipi         soft    nofile  1024 
uzipi         hard    nofile  65536 
uzipi         soft    stack   10240

检查/etc/pam.d/login，增加以下行，如果已经存在则不用增加（为什么要这样做？）： 

session required pam_limits.so

检查/etc/pam.d/su，增加以下行（为什么要这样做？）： 

session required pam_limits.so

oracle默认是不支持ubuntu，需要欺骗一下oracle安装程序：

mkdir /usr/lib64 
ln -s /etc /etc/rc.d 
ln -s /lib/x86_64-linux-gnu/libgcc_s.so.1 /lib64/ 
ln -s /usr/bin/awk /bin/awk 
ln -s /usr/bin/basename /bin/basename 
ln -s /usr/bin/rpm /bin/rpm 
ln -s /usr/lib/x86_64-linux-gnu/libc_nonshared.a /usr/lib64/ 
ln -s /usr/lib/x86_64-linux-gnu/libpthread_nonshared.a /usr/lib64/ 
ln -s /usr/lib/x86_64-linux-gnu/libstdc++.so.6 /lib64/ 
ln -s /usr/lib/x86_64-linux-gnu/libstdc++.so.6 /usr/lib64/ 
echo ‘Red Hat Linux release 5′ > /etc/redhat-release

6、建立oracle用户组及安装目录

使用独立的oracle用户安装oracle，首先这里创建oinstall与dba两个用户组：

$sudo groupadd oinstall
$sudo groupadd dba
$sudo mkdir -p /home/oracle

然后创建oracle用户：

$sudo useradd -g oinstall -G dba -s /bin/bash oracle
$sudo passwd oracle
$sudo chown -R oracle:oinstall /home/oracle

配置一部分环境变量。
打开.profile文件

$su
$vi .profile

不建议在安装oracle之前就配置环境变量，建议只配置以下变量：

# Oracle Settings
TMP=/tmp;
export TMP
TMPDIR=$TMP;
export TMPDIR

配置完后，直接生效：

$source .profile

二、安装oracle

1、下载oracle

到oracle官方下载oracle linux 64bit版，这里是下载地址。

2、解压缩oracle安装包

下载的格式为tar.gz，一般是下面两个压缩文件：

  • linux.x64_11gR2_database_1of2.zip

  • linux.x64_11gR2_database_2of2.zip

将压缩文件解压到某个目录，比如我解压在：/home/uzipi/Downloads/database下，然后赋予权限。

$cd /home/uzipi/Downloads/database/install
$chmod +x *.sh .oui unzip
$sudo chown -R oracle:oinstall /home/uzipi/Downloads/database

然后再执行下面的命令，打开安装界面GUI，开始安装。

$xhost + #使所有用户都能访问图形桌面
$cd /home/uzipi/Downloads/database
$chmod +x runInstaller
$./runInstaller

3、安装过程出现的问题

（1）安装过程到安装oraInventory这一步时，提示/home/oraInventory不是一个合法的目录“specify a valid inventory location”，原因是因为/home目录下面没有oraInventory目录。
我们在/home下创建一个oraInventory目录，并为其赋权限：

$su 
$mkdir /home/oraInventory
$chown -R oracle:oinstall /home/oraInventory

创建并赋权完毕后，继续next安装，成功进入到下一步。

进入到“Perform Prerequisite Checks”这一步时，有很多Check都没通过，但可以选择Ignore All跳过，开始安装。

因为我是安装在/home/oracle目录下，在安装读条的过程中，GUI警告提示了这个信息，yes忽略警告继续安装，当然在生产环境中，最好是建立在/opt/oracle目录下，避免对home目录的影响。

（2）遇到makefile的错误
另开一个终端，键入命令行：
sed -i ‘s/^(\s\$(MK_EMAGENT_NMECTL))\s$/\1 -lnnz11/g’ $ORACLE_HOME/sysman/lib/ins_emagent.mk

$ORACLE_HOME=/home/oracle/product/11.2.0/dbhome_1

对应的mk文件，修改上面的命令，继续执行，然后retry
sed -i ‘s/^(\s\$(MK_EMAGENT_NMECTL))\s$/\1 -lnnz11/g’ $ORACLE_HOME/sysman/lib/ins_rdbms.mk

sed -i ‘s/^(\s\$(MK_EMAGENT_NMECTL))\s$/\1 -lnnz11/g’ ins_rdbms.mk

配置oracle的环境变量：

sudo gedit ~/.profile

打开.profile文件，为文件中添加以下行： 

#Oracle
export ORACLE_HOSTNAME=localhost;
export ORACLE_BASE=/home/oracle;
export ORACLE_HOME=$ORACLE_BASE/product/11.2.0/dbhome_1;
export ORACLE_SID=orcl;
export PATH=$PATH:$ORACLE_HOME/bin/
export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:$ORACLE_HOME/lib
export NLS_LANG="Simplified chinese_china.al32utf8"
hello
